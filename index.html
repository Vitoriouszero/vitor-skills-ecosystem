ui-ux-pro-max (nextlevelbuilder) 		Por que juntar: As três têm o mesmo acionador ("cria algo visual em HTML") mas cada uma traz uma camada diferente:

• frontend-design = filosofia anti-genérica (COMO pensar o design)
• ui-ux-pro-max = banco de dados validado (COM QUÊ: 161 paletas, 24 padrões LP, 67 estilos, 99 regras UX)

Juntas formam: "gero uma LP com estrutura de conversão + visual distintivo + sistema de design correto para o tipo de produto." Separadas, qualquer uma das três pode ativar quando você quer as outras três também.

Sem perda porque: ui-ux-pro-max e frontend-design têm escopos não sobrepostos — uma é filosófica, outra é um banco de dados. Criar-landing-page é a âncora de conversão. As três se somam, nunca se repetem.

Nome mais amplo justificado: "web-builder" porque a skill poderá gerar LPs, pages de produto, páginas de captura, páginas de checkout — não só "landing page". O trigger se expande junto. 		Fusão 3→1 IMEDIATO
